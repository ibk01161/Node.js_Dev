#2. Node.js 개발 환경 구축


2.1. LTS 버전과 Current 버전

* Current 버전
- Current 버전은 '어떠한 변화가 일어나고 있는 상태의 버전', 조금 안정적이지 않을 수 있음
- Node.js는 6개월 단위로 버전 숫자를 하나 올림, 6개월마다 새로운 Current버전이 등장

* LTS 버전
- 짝수 버전을 가진 Current 버전은 6개월이 지나면 LTS버전으로 바뀜
- LTS 버전은 30개월의 장기적인 지원을 약속받은 버전이라는 의미
- LTS 버전은 버전의 숫자가 오른다고 해도 호환성을 깨뜨리는 변경 사항이 추가되지 않음, 실무에서 Node.js로 개발할 때는 LTS 버전을 사용할 것을 추천
- Current 버전의 장점은 LTS 버전에서 발생한 문제 해결 또는 필요한 기능 추가를 지속해서 해줌

2.2 윈도우 설치
- 다운로드 페이지에서 msi 파일 installer를 선택해 설치, 설치가 완료되면 컴퓨터를 종료하고 다시 시작한 후 cmd창 실행
- cmd창에 'node'를 입력하면 Node.js 코드를 입력할 수 있는 입력 화면이 나타남
- 이곳을 REPL (Read Eval Print Loop) 이라고 부르며, 한 줄씩 코드를 입력해 실행해볼 수 있음
- 컨트롤 + C 두 번 누르면 REPL을 빠져나올 수 있음

2.3. 우분투 설치
- Node.js의 이전 버전에서는 Node.js의 소스 코드를 다운로드하고 컴파일하는 과정을 거쳤는데, Node.js 4.x 버전부터는 패키지 매니저를 사용해 간편하게 설치할 수 있게 됨

2.4. 첫 번째 Node.js 애플리케이션 (node.baisc.js)
- Node.js 파일을 실행하려면 우선 Node.js 파일을 생성해야 함, 확장명이 js인 파일을 사용
- 파일을 저장하고, 콘솔 화면에서 cd 명령어를 사용해 생성한 파일이 있는 폴더로 이동 (node '파일이름' 명령을 사용해 생성한 코드를 실행) 

2.5. 두 번째 Node.js 애플리케이션 (node.server.js)
- 새로운 Node.js 파일을 생성 후 저장
- 파일을 실행 시키면 자동으로 종료되지 않음 (웹 서버가 만들어졌으므로)
- 웹 브라우저를 사용해 개인 컴퓨터 (127.0.0.1)의 52273번 포트에 접속
- 접속하면 Hello World 출력
